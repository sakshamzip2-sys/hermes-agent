# PROPOSALS.md — learned rules and self-modifications awaiting approval

Every learned rule, workflow refinement, or self-modification the loop wants to make goes HERE
first, versioned and reversible. Nothing here is a standing rule until the user approves it.
CLAUDE.md is NEVER auto-rewritten; high-confidence proven rules are proposed here and only
appended to a delimited, versioned CLAUDE.md section after explicit approval. No em dashes.

## Format

Each proposal: an id, the rule/change, the evidence it is based on, the risk if wrong, the
target (memory / skill / prompt / CLAUDE.md), and how to revert. Status: proposed | approved |
rejected | applied.
